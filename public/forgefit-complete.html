CardioFit Run / cycle programming
Workout builder

Define intervals with time and/or distance, pace or speed, heart-rate zone, cadence, and coaching notes. Save as a clinic template or schedule onto a patient calendar.

Training paces (race or goal)

Get threshold pace and Z1–Z5 pace bands (min/km) from either a recent result or a goal finish time at a target event. Estimated times at other distances use Riegel (k=1.06). Heuristic only — not the commercial McMillan product.

Steps & blocks

Each step: time/distance, pace band, intensity Open, Zone (1–5 as HR, pace, or power), or %VO2. Work/rest reps for uniform intervals; Ladder for changing work/rest each rung; Repeat pattern to repeat a multi-segment cycle. Drag blocks from the palette into the dashed area, or drag a whole row (start on the grip or empty padding—not inputs) to reorder.

Import from TrainingPeaks

TrainingPeaks files: Put the workout on the calendar → open Quick View → Export → download .zwo (Zwift / run or ride), .mrc / .erg (cycling power indoor), .fit / .fit.gz (planned workout download), or paste JSON with Structure. FIT imports map steps best-effort; complex repeat rules may need .zwo instead. For a full plan, import each key session, then either paste structures into lib/racePlanTemplates.js or use POST /api/calendar/batch with a sessions array from your own tooling.

In the portal: Settings → Portal secret → CardioFit → choose a file or paste JSON → import → review → Save as template or schedule. Pasted JSON: full plan object or Structure only; nested Repetition is expanded.

Strava

Patients connect Strava in Settings → Integrations (same user ID as Garmin). For completed calendar sessions, use Post to Strava on the Calendar day list — ForgeFit creates a manual Strava activity (time, distance from your plan when possible).

Apple Health & Watch

HealthKit and WorkoutKit are only available inside a native iOS app. Garmin workouts sync to Apple Health when the patient uses Garmin Connect; structured Apple Watch-only plans need a future ForgeFit iOS build or manual entry in the Watch workout app.

Load & adherence trends

Compare week-to-week planned vs completed load (TSS, time) and adherence for structured run/cycle sessions. Use alongside subjective readiness to spot fatigue or burnout patterns—signals are heuristic, not diagnoses.

Guidance uses recovery_data (readiness, sleep, HRV) plus rolling completed load. Optional URL param &ref=YYYY-MM-DD on the API sets the reference day (defaults to today UTC).
